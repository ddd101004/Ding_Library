本文档描述论文数据从外部 API 到本地数据库再到 FastGPT 知识库的完整同步流程。

## 三、FastGPT API 使用说明

### API 层级区分

| API 类型 | 用途 | 接口路径 |
|---------|------|---------|
| **Dataset API** | 创建/查询论文文件夹 | `/api/core/dataset/create`, `/api/core/dataset/list` |
| **Collection API** | 创建元数据/全文集合 | `/api/core/dataset/collection/create/*` |

### 正确的使用方式

1. **创建论文文件夹**：使用 `createDataset` (type: folder, parentId 指向公共数据集)
2. **查询论文文件夹**：使用 `listDatasets` (searchKey 搜索 intro 字段)
3. **创建元数据**：使用 `createTextCollection` (datasetId 指向论文文件夹)
4. **上传全文 PDF**：使用 `createFileCollection` (datasetId 指向论文文件夹)

## 四、FastGPT 存储结构

```
公共数据集 (FASTGPT_PUBLIC_DATASET_ID)
│
├── 📁 论文文件夹 1 (Dataset, type: folder)
│   │  _id: "folder-id-123"  ← 论文文件夹的 Dataset ID
│   │  name: "深度学习在自然语言处理中的应用研究"
│   │  intro: "edsjsr:10.1234/example.001"  ← sourceId，用于去重匹配
│   │
│   ├── 📄 元数据 (Collection, type: text)
│   │   datasetId: "folder-id-123"  ← 指向论文文件夹
│   │   parentId: null  ← 直接在 Dataset 根目录下
│   │   name: "元数据"
│   │   content: "# 论文标题\n## 作者\n## 摘要\n## 关键词..."
│   │
│   └── 📎 全文 (Collection, type: file) [后续补充]
│       datasetId: "folder-id-123"  ← 指向论文文件夹
│       parentId: null  ← 直接在 Dataset 根目录下
│       name: "全文"
│       file: PDF 文件
│
├── 📁 论文文件夹 2 (Dataset, type: folder)
│   │  ...
│
└── ...更多论文文件夹
```

### 关联机制说明

**核心原理**：Collection 通过 `datasetId` 字段与 Dataset 关联

| 层级 | 类型 | 关联方式 |
|------|------|---------|
| 公共数据集 | Dataset (type: dataset) | 根级，无 parentId |
| 论文文件夹 | Dataset (type: folder) | `parentId` 指向公共数据集 |
| 元数据/全文 | Collection | `datasetId` 指向论文文件夹，`parentId` 为 null |

**代码示例**：
```typescript
// 1. 创建论文文件夹（Dataset）
const folderId = await createDataset({
  parentId: FASTGPT_PUBLIC_DATASET_ID,  // 父级是公共数据集
  type: "folder",
  name: "论文标题",
  intro: sourceId,  // 用于去重搜索
});

// 2. 创建元数据（Collection）- 关联到论文文件夹
await createTextCollection({
  datasetId: folderId,  // 指向论文文件夹，建立关联
  parentId: null,       // 直接在 Dataset 根目录下
  name: "元数据",
  text: "论文内容...",
});

// 3. 创建全文（Collection）- 关联到论文文件夹
await createFileCollection({
  datasetId: folderId,  // 指向论文文件夹，建立关联
  parentId: null,       // 直接在 Dataset 根目录下
  file: pdfBuffer,
  filename: "全文",
});
```

### 命名规则

| 层级 | name | intro | 说明 |
|------|------|-------|------|
| 论文文件夹 | 论文标题 (截取前100字符) | sourceId | intro 用于 searchKey 去重 |
| 元数据集合 | `{论文标题}_元数据` | - | 包含论文标题便于识别 |
| 全文集合 | `{论文标题}_全文` | - | 包含论文标题便于识别 |

## 五、去重机制（两层去重）

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                            论文去重机制（两层去重）                                │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ┌──────────────────────────────────────────────────────────────────────────┐   │
│  │  第1层：数据库字段检查                                                     │   │
│  │  ─────────────────────────────────────────────────────────────────────── │   │
│  │  检查 Paper.fastgpt_collection_id 字段是否已有值                          │   │
│  │  ● 有值 → 跳过同步，直接返回已存在的 folderId                             │   │
│  │  ● 无值 → 进入第2层检查                                                   │   │
│  └──────────────────────────────────────────────────────────────────────────┘   │
│                                      │                                          │
│                                      ▼                                          │
│  ┌──────────────────────────────────────────────────────────────────────────┐   │
│  │  第2层：FastGPT 远程查询去重（使用 Dataset API）                           │   │
│  │  ─────────────────────────────────────────────────────────────────────── │   │
│  │  1. 调用 listDatasets API，传入参数：                                     │   │
│  │     - parentId: 公共数据集ID                                              │   │
│  │     - searchKey: sourceId (搜索 intro 字段)                               │   │
│  │  2. 遍历搜索结果，通过 intro 字段精确匹配 sourceId                         │   │
│  │  ● 找到匹配 → 更新本地数据库记录，返回已存在的 folderId                    │   │
│  │  ● 未找到   → 使用 createDataset 创建新的论文文件夹                        │   │
│  └──────────────────────────────────────────────────────────────────────────┘   │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 关键字段说明

| 字段 | 存储位置 | 说明 |
|------|---------|------|
| `source_id` | Paper 表 | 外部系统唯一标识 (如 EBSCO 的 "dbId:an") |
| `intro` | FastGPT 文件夹 | 存储 sourceId，用于 searchKey 搜索匹配 |
| `fastgpt_collection_id` | Paper 表 | FastGPT 中的文件夹 ID（实际存储 Dataset ID）|

## 九、全文更新流程

**文件**: `src/service/fastgpt/publicDataset.ts`
**函数**: `updatePaperFulltextInDataset()`

```
触发条件: 论文 PDF 下载完成（如全文传递完成）
     │
     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ updatePaperFulltextInDataset(paperId)                                   │
└─────────────────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────┐
│ 前置检查             │    ● fastgpt_collection_id 是否存在（已同步元数据）
│                     │    ● fastgpt_has_fulltext 是否为 false（未同步全文）
│                     │    ● pdf_downloaded 是否为 true（PDF 已下载）
│                     │    ● pdf_file_path 是否存在（COS 路径）
└─────────────────────┘
     │
     ▼ 检查通过
┌─────────────────────┐
│ 从 COS 下载 PDF     │    downloadFile() → 临时文件
└─────────────────────┘
     │
     ▼
┌─────────────────────┐
│ 上传到 FastGPT       │    createFileCollection()
│                     │    parentId: paper.fastgpt_collection_id (文件夹 ID)
│                     │    与元数据并列存放在文件夹下
└─────────────────────┘
     │
     ▼
┌─────────────────────┐
│ 更新 Paper 表        │    fastgpt_has_fulltext = true
└─────────────────────┘
     │
     ▼
┌─────────────────────┐
│ 清理临时文件         │    删除下载的临时 PDF
└─────────────────────┘
```

### 全文更新说明

更新全文时**无需再次搜索 FastGPT**，文件夹 ID 已存储在本地数据库：

```
Paper 表
├── id: "uuid-xxx"
├── fastgpt_collection_id: "folder-id-123"  ◀── 首次同步时存储
├── fastgpt_has_fulltext: false
└── pdf_file_path: "papers/xxx.pdf"
```

### 批量更新全文

**函数**: `batchUpdatePaperFulltext(limit, delay)`

查询条件：
- `fastgpt_collection_id IS NOT NULL` (已同步元数据)
- `fastgpt_has_fulltext = false` (未同步全文)
- `pdf_downloaded = true` (PDF 已下载)
- `pdf_file_path IS NOT NULL` (COS 路径存在)
- `deleted_status = 0` (未删除)

## 十、核心数据库字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `source` | String | 数据来源：ebsco, aminer, wanfang |
| `source_id` | String | 来源系统唯一标识 (如 EBSCO 的 "dbId:an") |
| `fastgpt_collection_id` | String? | FastGPT 中的文件夹 ID |
| `fastgpt_sync_source` | String? | 同步来源：cron_task, user_search, chat_related |
| `fastgpt_sync_time` | DateTime? | 首次同步时间 |
| `fastgpt_has_fulltext` | Boolean | 是否已同步全文 (默认 false) |
| `pdf_downloaded` | Boolean | PDF 是否已下载 |
| `pdf_file_path` | String? | COS 中的 PDF 路径 |
