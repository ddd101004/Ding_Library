# 本地文件存储使用指南

本文档介绍如何使用本地文件系统存储论文和封面图。

## 架构说明

```
文件上传流程：
前端 → 后端 API → 本地文件系统 → 返回文件路径 → 保存到数据库

文件访问流程：
前端 → 静态文件服务 (/uploads) → 本地文件系统
```

## 目录结构

```
project/
├── uploads/                  # 文件存储目录
│   ├── papers/              # 论文文件
│   │   └── {user_id}/       # 按用户分目录
│   │       └── {timestamp}_{random}_{userId}.pdf
│   └── covers/              # 封面图片
│       └── {user_id}/
│           └── {timestamp}_{random}_{userId}.jpg
```

## 后端 API 使用

### 1. 上传论文

**接口**: `POST /api/upload/paper`

**请求类型**: `multipart/form-data`

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 论文文件（PDF/DOC/DOCX/TXT） |
| title | string | 否 | 论文标题 |
| authors | string | 否 | 作者信息 |
| abstract | string | 否 | 摘要 |
| keywords | string | 否 | 关键词 |

**返回示例**:
```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "paper_id": "uuid",
    "file_path": "papers/user123/1234567890_a1b2c3d4_user123.pdf",
    "file_url": "/uploads/papers/user123/1234567890_a1b2c3d4_user123.pdf",
    "file_name": "论文标题.pdf",
    "file_size": 2048576
  }
}
```

### 2. 上传封面图

**接口**: `POST /api/upload/cover`

**请求类型**: `multipart/form-data`

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 图片文件（JPG/PNG，最大 2MB） |

**返回示例**:
```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "file_path": "covers/user123/1234567890_x1y2z3w4_user123.jpg",
    "file_url": "/uploads/covers/user123/1234567890_x1y2z3w4_user123.jpg",
    "file_name": "cover.jpg",
    "file_size": 102400
  }
}
```

## 前端使用示例

### 上传论文

```typescript
import { uploadPaper, validateFileType, validateFileSize, formatFileSize } from "@/utils/uploadFile";

// 示例：上传按钮点击处理
const handleUploadPaper = async (file: File) => {
  // 验证文件类型
  if (!validateFileType(file, [".pdf", ".doc", ".docx", ".txt"])) {
    alert("不支持的文件类型");
    return;
  }

  // 验证文件大小（100MB）
  if (!validateFileSize(file, 100)) {
    alert("文件大小不能超过 100MB");
    return;
  }

  try {
    // 调用上传接口
    const result = await uploadPaper({
      file,
      title: "我的论文",
      authors: "张三",
      abstract: "论文摘要",
      keywords: "关键词1,关键词2",
    });

    console.log("上传成功", result);
    // result.file_url 可以用于访问文件
  } catch (error) {
    console.error("上传失败", error);
  }
};
```

### 上传封面图

```typescript
import { uploadCover } from "@/utils/uploadFile";

const handleUploadCover = async (file: File) => {
  try {
    const result = await uploadCover(file);
    console.log("封面图上传成功", result);
    // 保存 result.file_path 到数据库
  } catch (error) {
    console.error("上传失败", error);
  }
};
```

### React 组件示例

```tsx
import { useState } from "react";
import { uploadPaper, validateFileType, validateFileSize, formatFileSize } from "@/utils/uploadFile";

export function PaperUploadForm() {
  const [uploading, setUploading] = useState(false);
  const [progress, setProgress] = useState(0);

  const handleFileChange = async (e: React.ChangeEvent<HTMLInputElement>) => {
    const file = e.target.files?.[0];
    if (!file) return;

    // 验证
    if (!validateFileType(file, [".pdf"])) {
      alert("仅支持 PDF 文件");
      return;
    }

    if (!validateFileSize(file, 100)) {
      alert(`文件大小不能超过 100MB，当前文件：${formatFileSize(file.size)}`);
      return;
    }

    // 上传
    setUploading(true);
    try {
      const result = await uploadPaper({ file });
      alert("上传成功！");
      console.log("文件访问URL:", result.file_url);
    } catch (error: any) {
      alert("上传失败: " + error.message);
    } finally {
      setUploading(false);
    }
  };

  return (
    <div>
      <input
        type="file"
        accept=".pdf"
        onChange={handleFileChange}
        disabled={uploading}
      />
      {uploading && <p>上传中...</p>}
    </div>
  );
}
```

## 文件存储说明

### 数据库存储内容

数据库 **不存储** 文件二进制数据，**仅存储** 文件路径引用：

```sql
-- user_uploaded_papers 表
file_path = "papers/user123/1234567890_a1b2c3d4_user123.pdf"

-- paper_folders 表
cover_image = "covers/user123/1234567890_x1y2z3w4_user123.jpg"
```

### 访问文件

通过 URL 访问：`/uploads/{file_path}`

例如：
- 论文: `http://localhost:3007/uploads/papers/user123/1234567890_a1b2c3d4_user123.pdf`
- 封面: `http://localhost:3007/uploads/covers/user123/1234567890_x1y2z3w4_user123.jpg`

### 文件大小限制

| 文件类型 | 大小限制 |
|---------|---------|
| 论文文件 | 100MB |
| 封面图片 | 2MB |
| 用户头像 | 2MB |

### 支持的文件类型

| 文件类型 | 支持格式 |
|---------|---------|
| 论文 | .pdf, .doc, .docx, .txt |
| 封面 | .jpg, .jpeg, .png |
| 头像 | .jpg, .jpeg, .png, .gif, .webp |

## 从 COS 迁移到本地存储（可选）

如果你之前使用腾讯云 COS，现在想迁移到本地存储，可以按照以下步骤：

1. **创建上传目录**:
   ```bash
   mkdir -p uploads/papers
   mkdir -p uploads/covers
   ```

2. **下载 COS 文件**:
   - 使用腾讯云 COS 控制台或 API 下载所有文件
   - 保存到 `uploads/` 对应目录

3. **更新数据库路径**:
   ```sql
   -- 将 cos_key 转换为本地路径
   UPDATE user_uploaded_papers
   SET file_path = CONCAT('papers/', user_id, '/', SUBSTRING_INDEX(file_path, '/', -1))
   WHERE source = 'cos';

   -- 将封面图路径转换
   UPDATE paper_folders
   SET cover_image = CONCAT('covers/', user_id, '/', SUBSTRING_INDEX(cover_image, '/', -1))
   WHERE cover_image LIKE 'covers/%';
   ```

4. **测试文件访问**:
   - 访问 `/uploads/papers/xxx.pdf` 确认文件可访问
   - 检查前端显示是否正常

## 注意事项

1. **备份**: 定期备份 `uploads/` 目录和数据库
2. **磁盘空间**: 监控服务器磁盘空间使用情况
3. **权限**: 确保 `uploads/` 目录有正确的读写权限
4. **安全**: 文件访问需要登录认证（通过 `withAuth` 中间件）
5. **清理**: 定期清理无用的上传文件
